# 项目结构

本文档定义 Open Cluster Claw 前端项目的目录组织和文件布局。

## 目录结构

```
web/
├── src/
│   ├── components/           # 通用组件
│   │   ├── layout/          # 布局组件
│   │   └── common/          # 业务通用组件
│   ├── pages/               # 页面组件
│   ├── services/            # API 服务
│   ├── hooks/               # 自定义 Hooks
│   ├── types/               # TypeScript 类型定义
│   ├── lib/                 # 工具函数
│   ├── i18n/                # 国际化配置
│   │   ├── locales/         # 语言包
│   │   │   ├── zh-CN.ts     # 简体中文
│   │   │   └── en-US.ts     # 英文
│   │   └── index.ts         # i18n 初始化
│   ├── providers/           # Context Providers
│   │   ├── ThemeProvider.tsx
│   │   └── I18nProvider.tsx
│   ├── styles/              # 全局样式
│   ├── App.tsx              # 根组件
│   └── main.tsx             # 入口文件
├── package.json
└── vite.config.ts
```

## 目录说明

### components/
- `layout/`：布局相关组件，如 Header、Sidebar、Footer
- `common/`：业务通用组件，如 LanguageSwitcher、ThemeSwitcher

### pages/
页面级组件，与路由一一对应。每个页面组件使用 `export default`。

### services/
API 服务封装，按资源分组（instanceApi、configApi 等）。

### hooks/
自定义 React Hooks，提取可复用的状态逻辑。

### types/
TypeScript 类型定义，集中管理接口类型。

### lib/
通用工具函数。

### i18n/
国际化配置和语言包文件。

### providers/
React Context Providers，如主题、国际化。

### styles/
全局样式和 CSS 变量定义。
